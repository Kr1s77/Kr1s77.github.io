限流视图说明

1. 两个基类

视图中有两个基类:

  1. APIView
  2. GenericAPIView

1）APIView

以前我们定义视图函数继承的都是 View 类.

但是 Django REST Framework 使用的是 APIView 类

APIView 是 DRF 框架所有视图的基类，继承自 Django 的 View 父类

# 使用方式: 

# 导入
from rest_framework.views import APIView

# 对应类视图,继承自 APIView
class 类视图名(APIView):
    pass

APIView 与 View 对比:

APIView 与 View 的不同之处在于：

  • 接收的 request 对象不同:
    • 如果类视图继承自 APIView , 则类视图中的函数, 接收的 request 对象, 都是 Request 类对象
    • 而 View 中的为: HttpResponse 对象.
  • 返回的 reponse 对象不同:
    • 如果类视图继承自 APIView , 视图函数返回的也是 DRF 的 Response 对象
    • 而 View 中的是 HttpResponse 对象.
  • 任何 APIException 异常都会被捕获到，并且处理成对应的响应信息.
  • 在进行 dispatch( ) 分发前，会对请求进行身份认证、权限检查、流量控制等.

注意:

在 APIView 中仍以常规的类视图定义方法来实现 get() 、post( ) 或者其他请求方式的方法

展示

我们上面已经学习了 Response 类 和 Request 类, 以及 序列化器类

那么我们现在就把这个些给结合起来, 一起进行使用:

用什么结合呢? 使用 APIView

定义一个类视图 只要继承自 APIView, 则它内部可以不但可以使用 Response 和 Request, 还可以使用 序列化器:

例如:

这是我们以前写的 RESTful 风格代码:

class BookListView(View):
    def get(self, request):
        queryset = BookInfo.objects.all()
        book_list = []
        for book in queryset:
            book_list.append({
                'id': book.id,
                'btitle': book.btitle,
                'bpub_date': book.bpub_date,
                'bread': book.bread,
                'bcomment': book.bcomment,
                'image': book.image.url if book.image else ''
            })
        return JsonResponse(book_list, safe=False)

现在就可以变为: 

from rest_framework.views import APIView
from rest_framework.response import Response

# url(r'^books/$', views.BookListView.as_view()),
# 我们定义一个类视图, 继承自 APIView
class BookListView(APIView):
    # 添加 get 等函数
    def get(self, request):
        # 获取查询集:
        books = BookInfo.objects.all()
        # 调用序列化器, 对查询集进行序列化处理
        serializer = BookInfoSerializer(books, many=True)
        # 处理完成的数据, 再经过 Response 类的处理就会变成 json
        return Response(serializer.data)

2）GenericAPIView

这节我们来介绍另一个 DRF 框架基类: GenericAPIView

# 使用方式: 

# 导入
from rest_framework.generics import GenericAPIView

# 对应类视图,继承自 GenericAPIView
class 类视图名(GenericAPIView):
    pass

继承自 APIVIew

在 APIView 的基础上主要增加了 操作序列化器 和 数据库查询的方法

作用是为下面Mixin扩展类的执行提供方法支持。

通常在使用时，可搭配一个或多个Mixin扩展类。

GenericAPIView 扩展的功能:

功能一: serializer_class

serializer_class 属性用于指定使用的序列化器

一般我们可以通过这个属性指定 需要使用的 序列化器

使用方式: 


class BookListView(GenericAPIView):
    # 指明当前视图使用 BookInfoSerializer 
    # 这个序列化器类来进行数据的序列化
    serializer_class = BookInfoSerializer

这个属性一旦指定之后, 我们就可以调用和这个属性相关的函数 :

1. get_serializer_class( )

可以通过这个函数, 获取我们通过这个属性: serializer_class = HeroInfoSerializer 定义的序列化器类, 即 HeroInfoSerializer

使用方式: 


class BookListView(GenericAPIView):
    # 这个属性下面会讲,这里暂时不提:
    queryset = BookInfo

    # 指明当前视图使用 HeroInfoSerializer 
    # 这个序列化器类来进行数据的序列化
    serializer_class = BookInfoSerializer

    def get(self, request):
        # 获取当前类中定义的序列化器类, 即上面的 HeroInfoSerializer
        className = get_serializer_class()
        print(className)
        return Response('get func')

结果: 

<class 'booktest.serializers.BookInfoSerializers'>
2. get_serializer( )

作用: 可以通过这个函数获取我们的序列化器对象

即: 根据字面意思来理解即可.

使用方式: 


class BookListView(GenericAPIView):
    # 这个属性下面会讲,这里暂时不提:
    queryset = BookInfo

    # 指明当前视图使用 HeroInfoSerializer 
    # 这个序列化器类来进行数据的序列化
    serializer_class = BookInfoSerializer

    def get(self, request):
        # 获取当前类中定义的序列化器对象, 即上面的 HeroInfoSerializer类 所生成的对象
        # 也就是:  obj = HeroInfoSerializer( ) 这里的obj
        serializerObj = self.get_serializer()
        print(serializerObj)

        # 我是快乐的分隔符~
        print('-------------------------------')

        # 我们获取一下序列化器类对象, 和上面的结果进行验证, 会发现是一样的
        obj = BookInfoSerializers()
        print(obj)
        return Response('get func')

    • format 当前请求期望返回的数据格式

功能二: queryset

这个属性用于设置我们需要序列化的模型类 (即: 数据查询集)

使用方式: 

from rest_framework.generics import GenericAPIView
from booktest.models import BookInfo
from booktest.serializers import BookInfoSerializers 
from django.http import HttpResponse


class BookListView(GenericAPIView):
    # 指明当前视图使用 HeroInfoSerializer 
    # 这个序列化器类来进行数据的序列化
    serializer_class = HeroInfoSerializer

    # 设置当前类视图中所用的模型类 (即: 数据查询集)
    queryset = BookInfo

和这个属性相关的, 也有一些方法, 我们下面来一起学习:

1. get_queryset( )

返回视图使用的查询集( 我们当前视图所使用的模型类 )

主要用来提供给Mixin扩展类使用

是Mixin扩展类中 列表视图 与 详情视图 获取数据的基础

使用方式: 


class BookListView(GenericAPIView):
    # 给当前类视图添加模型类 (查询集)
    queryset = BookInfo
    serializer_class = BookInfoSerializer

    def get(self, request):
         # 获取模型类(查询集), 就是上面 queryset = BookInfo 里面的 BookInfo
        queryset = self.get_queryset()
        print(queryset)
        return Response('get func')

2. get_object( )

可以通过这个方法, 获取对应的某一个模型类对象

主要用来提供给Mixin扩展类使用

视图部分:

views.py

使用方式: 


class BookListView(GenericAPIView):
    # 给当前类视图添加模型类 (查询集)
    queryset = BookInfo
    serializer_class = BookInfoSerializer

    # 这里注意, 我们和上面是不一样的, 这里的参数, 多了一个pk
    # 如果没有传入 pk ,则当前函数(get_object)会报错.
    def get(self, request, pk):
         # 获取 pk 这个 key 所对应的模型类对象.
        object = self.get_object()
        print(object)
        return Response('get func')

路由: 

urlpatterns = [
    url(r'^books/(?P<pk>\d+)/$', views.BookListView.as_view())
]

该方法会默认使用 APIView 提供的 check_object_permissions 方法检查当前对象是否有权限被访问

我们已经学习完了 GenericAPIView的属性和方法, 我们现在就可以试着将原来写的接口进行修改:

import json
from rest_framework.views import APIView
from rest_framework.generics import GenericAPIView
from rest_framework.response import Response
from rest_framework import status
from django.http import Http404
from booktest.models import BookInfo, HeroInfo
from booktest.serializers import BookInfoSerializer


class BookListView(GenericAPIView):
    # 指定当前视图使用查询集
    queryset = BookInfo.objects.all()
    # 指定当前视图使用序列化器类
    serializer_class = BookInfoSerializer

    def get(self, request):
        """获取所有图书信息"""
        # 查询出所有图书信息
        obj = self.get_queryset()

        # 返回所有图书信息
        serializer = self.get_serializer(obj, many=True)

        # return json
        return Response(serializer.data)

    def post(self, request):
        """新建一本图书信息"""
        # 前端传递参数时，使用json传递
        # 反序列化-数据校验
        serializer = self.get_serializer(data=request.data)
        serializer.is_valid(raise_exception=True)

        # 反序列化-数据保存(create)
        serializer.save()

        # 返回响应: status: 201，包含新建图书的信息
        return Response(serializer.data, status=status.HTTP_201_CREATED)

# /books/10/
class BookDetailView(GenericAPIView):
    # 指定当前视图使用查询集
    queryset = BookInfo.objects.all()

        """获取指定的图书信息"""
        # 根据pk查询指定图书信息
        obj = self.get_object()

        # 返回响应数据
        serializer = self.get_serializer(obj)
        return Response(serializer.data)

    def put(self, request, pk):
        """修改指定的图书信息"""
        # 根据pk查询指定图书
        obj = self.get_object()

        # 前端传递参数时，使用json传递
        # 获取参数(btitle, bpub_date)并进行参数校验
        # 反序列化-数据校验
        serializer = self.get_serializer(obj, data=request.data)

        # 反序列化-数据保存(update)
        serializer.save()

        # 返回响应数据
        return Response(serializer.data)

    def delete(self, request, pk):

        # 根据pk查询指定图书
        obj = self.get_object()

        obj.delete()

        return Response(status=status.HTTP_204_NO_CONTENT)

2. 五个扩展类

作用：

提供了几种后端视图（对数据资源进行曾删改查）处理流程的实现

如果使用这五种扩展类，则视图可以通过继承相应的方法来减少自己编写的代码量。

这五个扩展类都需要搭配 GenericAPIView 父类，因为五个扩展类的实现需要调用 GenericAPIView 提供的序列化器与数据库查询的方法

1）ListModelMixin

列表视图扩展类:

作用:

  • 提供 list(request, args, *kwargs) 方法来快速实现列表视图
  • 返回200状态码

ListModelMixin: 这个扩展类能够给我们提供一个 list( ) 函数

这个函数其实是对我们以前做的操作进行了封装. 如下所示, 是它封装的内容.

这个函数大家可以这样理解: 

def list(self, request):
    # 查询出所有图书信息
    books = BookInfo.objects.all()

    # 序列化所有图书信息, 得到 json 字符串
    serializer = BookInfoSerializer(books, many=True)

    # 创建 response 对象, 并返回
    response = HttpResponse(serializer.data)

    # 返回 response 对象
    return response

也可以理解为: 

def list(self, request):
    # 查询出所有图书信息
    obj = self.get_queryset()

    # 序列化所有图书信息, 得到 json 字符串
    serializer = self.get_serializer(obj, many=True)

    # 创建 response 对象, 并返回
    response = Response(serializer.data)

    # 返回 response 对象
    return response

其实上面的代码都是一样的, 一个是通过序列化器的方式来获取, 一个是通过 DRF 框架的方式来获取, 总之, ListModelMixin 帮我们把这些代码进行了封装. 以后我们调用的时候,这样使用就可以了:

例如:

# 导入(我这里只导入了一部分)
from rest_framework.generics import GenericAPIView
from rest_framework.mixins import ListModelMixin

# 我们定义一个视图类, 使其继承自 ListModelMixin 和 GenericAPIView
class BookListView(ListModelMixin, GenericAPIView):
    # 指定当前类视图的 查询集 为 BookInfo.objects.all() 
    queryset = BookInfo.objects.all()
    # 指定当前类视图所使用的序列化器是 BookInfoSerializer
    serializer_class = BookInfoSerializer

    # 定义 get 函数, 
    def get(self, request):
        # 获取所有图书信息, 并且以 json 形式返回
        # 这里调用的 list 就是 ListModelMixin 这个类的 list() 函数
        return self.list(request)

ListModelMixin的list 中的 list 方法还会对数据进行 过滤 和 分页, 我们可以通过查看源码来了解:

源代码：

class ListModelMixin(object):

    def list(self, request, *args, **kwargs):
        # 过滤
        queryset = self.filter_queryset(self.get_queryset())
        # 分页
        page = self.paginate_queryset(queryset)
        if page is not None:
            serializer = self.get_serializer(page, many=True)
            return self.get_paginated_response(serializer.data)
        # 序列化
        serializer = self.get_serializer(queryset, many=True)

        return Response(serializer.data)

2）CreateModelMixin

创建视图扩展类:

作用:

  • 提供 create(request, args, *kwargs) 方法来快速实现列表视图
  • 返回 201 状态码
  • 如果序列化器对前端发送的数据验证失败，返回400错误。
  • 在使用这个扩展类时, 需要在序列化器中定义 create 的序列化方法, 否则会报错
# 在序列化器中 添加如下函数:

def create(self, validated_data):
    return BookInfo.objects.create(**validated_data)

class CreateModelMixin(object):
    """
    Create a model instance.
    """
    def create(self, request, *args, **kwargs):
        # 获取序列化器
        serializer = self.get_serializer(data=request.data)
        # 验证
        serializer.is_valid(raise_exception=True)
        # 保存
        self.perform_create(serializer)
        headers = self.get_success_headers(serializer.data)
        return Response(serializer.data, status=status.HTTP_201_CREATED, headers=headers)

    def perform_create(self, serializer):
        serializer.save()

    def get_success_headers(self, data):
        try:
            return {'Location': str(data[api_settings.URL_FIELD_NAME])}
        except (TypeError, KeyError):
            return {}

class RetrieveModelMixin(object):

    Retrieve a model instance.
    """

        return Response(serializer.data)

4）UpdateModelMixin

更新视图扩展类

作用:

  • 提供 update(request, args, *kwargs) 方法来快速实现更新一个存在的数据对象
  • 同时也提供 partial_update(request, args, *kwargs) 方法, 可以实现局部更新
  • 返回200状态码
  • 序列化器校验数据失败时，返回400错误
  • 在使用这个扩展类时, 需要在序列化器中定义 update( ) 的序列化器方法,否则会报错

def update(self, instance, validated_data):
    instance.btitle = validated_data.get('btitle'),
    instance.bpub_date = validated_data.get('bpub_date'),
    instance.save()
    return instance

class UpdateModelMixin(object):
    """
    Update a model instance.
    """
    def update(self, request, *args, **kwargs):
        partial = kwargs.pop('partial', False)
        instance = self.get_object()
        serializer = self.get_serializer(instance, data=request.data, partial=partial)
        serializer.is_valid(raise_exception=True)
        self.perform_update(serializer)

        if getattr(instance, '_prefetched_objects_cache', None):
            # If 'prefetch_related' has been applied to a queryset, we need to
            # forcibly invalidate the prefetch cache on the instance.
            instance._prefetched_objects_cache = {}

        return Response(serializer.data)

    def perform_update(self, serializer):
        serializer.save()

    def partial_update(self, request, *args, **kwargs):
        kwargs['partial'] = True
        return self.update(request, *args, **kwargs)

class DestroyModelMixin(object):
    """
    Destroy a model instance.
    """
    def destroy(self, request, *args, **kwargs):
        instance = self.get_object()
        self.perform_destroy(instance)
        return Response(status=status.HTTP_204_NO_CONTENT)

    def perform_destroy(self, instance):
        instance.delete()
