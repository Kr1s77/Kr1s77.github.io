Introduction

QQ登录

学习目标:

  • 知道 QQ 登录的步骤

作用:

  • 能够使用登录页面进行第三方( QQ )登录

QQ 登录，亦即我们所说的第三方登录

是指用户可以不在本项目中输入密码，而直接通过第三方的验证，成功登录本项目。

若想实现QQ登录，需要成为QQ互联的开发者，审核通过才可实现:

注册方法可参考链接http://wiki.connect.qq.com/%E6%88%90%E4%B8%BA%E5%BC%80%E5%8F%91%E8%80%85

成为QQ互联开发者后，还需创建应用

即获取本项目对应与 QQ 互联的应用 ID，创建应用的方法参考链接http://wiki.connect.qq.com/__trashed-2

QQ 登录开发文档连接> http://wiki.connect.qq.com/%E5%87%86%E5%A4%87%E5%B7%A5%E4%BD%9C_oauth2-0

QQ登录的流程

创建新的子应用 ( oauth )

创建一个新的应用oauth，用来实现QQ第三方认证登录。

python  ../../manage.py  startapp  oauth

配置总路由 oauth/

# TODO 6.1 配置 oauth 子应用的总路由
url(r'^oauth/', include('oauth.urls'))

我们定义一个模型的基类, 用与第三方登录:

创建  utils / BaseModels.py 文件

创建模型类基类，用于增加数据新建时间和更新时间。

from django.db import models

# TODO 6.2 在新建的模型类中添加一个模型基类:
# 我们创建一个模型类的基类: BaseModel
class BaseModel(models.Model):
    """为模型类补充字段"""
    # 添加创建时间字段, 该字段会自动生成创建时间
    create_time = models.DateTimeField(auto_now_add=True, verbose_name="创建时间")
    # 添加更新字段, 该字段会自动添加更新时间
    update_time = models.DateTimeField(auto_now=True, verbose_name="更新时间")

    class Meta:
        # 说明该类是抽象模型类
        # 抽象类用于继承使用
        # 数据库迁移时不会创建BaseModel的表
        abstract = True

在 oauth / models.py 中定义QQ身份（openid）与用户模型类User的关联关系

from django.db import models
from meiduo_mall.utils.models import BaseModel

# TODO 6.3 基于模型基类, 我们创建一个新的模型类, 保存 qq 登录用户数据
class OAuthQQUser(BaseModel):
    """
    QQ登录用户数据
    """
    user = models.ForeignKey('users.User', on_delete=models.CASCADE, verbose_name='用户')
    # db_index = True 设置数据库索引
    openid = models.CharField(max_length=64, verbose_name='openid', db_index=True)

    class Meta:
        db_table = 'tb_oauth_qq'
        verbose_name = 'QQ登录用户数据'
        verbose_name_plural = verbose_name

进行数据库迁移

# 生成迁移文件
python manage.py makemigrations

python manage.py migrate

urllib 使用说明

在后端接口中，我们需要向QQ服务器发送请求，查询用户的QQ信息

其中, 用到一些我们之前没有接触的内容, 特在此做个说明:

Python 提供了一个标准模块 urllib

可以帮助我们快速的对一些内容进行处理

例如:

  • urllib.parse.urlencode( dict )

    将字典转换为查询字符串

    例如: 

    # 导入 python 中的 urllib 中的 urlencode:
from urllib.parse import urlencode

# 创建一个 dict 类:
data = {
    'id':1,
    'username':'zhangsan'
}
# 使用导入的函数, 进行转换:
res = urlencode(data)
# 打印结果:
print(res)

    结果:

    id=1&username=zhangsan

  • urllib.parse.parse_qs( )

    将查询字符串数据转换为python中的字典

    例如:

    # 导入 parse_qs 函数:
from urllib.parse import parse_qs
# 定义一个查询字符串格式的数据
data = 'a=1&b=2&c=3'
# 导入函数中:
result = parse_qs(data)
# 打印:
print(result)

    结果:

    {'a': ['1'], 'b': ['2'], 'c': ['3']}

    注意: key 对应的 value 是一个 list

  • urllib.request.urlopen( url, data=None)

    发送http请求:

    • 如果 data 为 None，发送 GET 请求
    • 如果 data 不为 None，发送 POST 请求

    返回 response 响应对象，可以通过 read( ) 读取响应体数据

    需要注意读取出的响应体数据为bytes类型

    # 导入 urlopen 函数:
from urllib.request import urlopen

# 定义 url
url = 'https://suggest.taobao.com/sug?code=utf-8&q=%E7%94%B5%E8%84%91'
# 调用函数, 发送请求
response = urlopen(url)
# 调用 read() 读取结果:
result = response.read()
# 打印结果:
print(result)

# 对结果进行解码:
res = result.decode()
# 再次打印结果:
print(res)

itsdangerous 的讲解

使用 itsdangerous 生成凭据 access_token

itsdangerous 模块的参考资料:

http://itsdangerous.readthedocs.io/en/latest/

安装
pip install itsdangerous
使用:
使用方法:
# 导入:
from itsdangerous import TimedJSONWebSignatureSerializer

# 创建对象
obj = TimedJSONWebSignatureSerializer(秘钥, 有效期(秒))


# 1. 生成 token
# 调用对象的 dumps( ) 方法, 传入数据
token_bytes = obj.dumps({'mobile': '18512345678'})
# 返回 bytes 类型, 我们可以对其解码,得到 token
token = token_bytes.decode()



# 2. 检验 token
# 调用对象的 loads( ) 函数来进行检验
# 验证失败，会抛出itsdangerous.BadData异常
data = obj.loads(token)

例如:

使用 itsdangerous 中封装好的 TimedJSONWebSignatureSerializer

获取带有有效期的 token

生成:

# 导入:
from itsdangerous import TimedJSONWebSignatureSerializer as Serializer

SECRET_KEY = 'y(qv191i(y74@=k5^*df+i1=ic2#@yr@*^aew1&_38e8!zu%rh'
# serializer = Serializer(秘钥, 有效期秒)
# 创建对象
serializer = Serializer(SECRET_KEY, 300)

# serializer.dumps(数据), 返回bytes类型
# 调用 dumps() 函数:
token = serializer.dumps({'mobile': '18512345678'})
token = token.decode()
print(token)
