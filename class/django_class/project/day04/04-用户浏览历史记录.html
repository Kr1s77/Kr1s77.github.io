Introduction

用户浏览历史记录

接下来我们要做的功能是这里的用户浏览记录功能:

浏览记录存储的相关思考:

  1. 用户在访问每个商品详情页面时，都要记录成浏览历史记录
  2. 我们把商品的浏览记录存储在 redis 中, 而不是存储在 mysql 里. 因为用户会频繁的访问商品, 存储到 mysql 中会造成数据库的效率下降.
  3. 浏览记录会存储商品的 sku_id .
  4. 我们会采用 redis 中的 列表来保存商品信息.( 因为需要保存用户浏览的顺序, 并按照这个顺序展示出来, 另外还不想过多的操作字符串, 所以综上所述, 还是使用列表最为合适.)

在 redis 中保存用户浏览记录的格式: 

'history_用户id': [sku_id列表]

在配置文件中增加浏览历史记录的 redis 配置

CACHES = {
    ...
    # 在 dev.py 的配置文件中添加一个新的存储区域
    # 用于保存用户的浏览记录: 
    # 存储区域的名称为: history
    "history": {
        "BACKEND": "django_redis.cache.RedisCache",
        "LOCATION": "redis://127.0.0.1:6379/4",
        "OPTIONS": {
            "CLIENT_CLASS": "django_redis.client.DefaultClient",
        }
    },
}

后端实现

在 users / serializes.py 中实现序列化器

# 这里我们定义的序列化器需要继承自: serializers.Serializer
# 因为我们定义的模型类中没有 sku_id 这个字段
# 所以继承自 ModelSerializer 没有作用.故. 我们这里继承自Serializer 
# 自己添加一个 sku_id 字段.
class AddUserBrowsingHistorySerializer(serializers.Serializer):
    """
    添加用户浏览历史序列化器
    """
    # 自定义一个商品编号的字段.最小值为1
    sku_id = serializers.IntegerField(label="商品SKU编号", min_value=1)

    # 对用户传入的 sku_id 进行检验: 
    def validate_sku_id(self, value):
        try:
            # 判断用户传入的 sku_id 是否在 SKU表 中有值.
            SKU.objects.get(id=value)
        except SKU.DoesNotExist:
            # 如果没有值, 则报错:
            raise serializers.ValidationError('该商品不存在')
        # 如果有, 就继续
        return value

    # 重写保存的 create 函数, 我们这里不是保存到 mysql 数据库, 
    # 而是需要保存到 redis 中去. 故,需要重写该方法:
    def create(self, validated_data):
        # 根据 redis 的链接方法, 获取存储区域为 history 的链接对象
        redis_conn = get_redis_connection("history")
        # 得到链接对象后, 调用 pipline() 函数,得到管道对象
        pl = redis_conn.pipeline()

        # 这里需要额外注意: 
        # 我们现在是在序列化器中, 在序列化器中想要得到当前的验证用户, 我们需想点办法
        # 首先我们应该思考, 一般验证用户从哪里能够找到---> 接口
        # 所以我们可以去上面的接口位置, 通过 request.user 得到
        # 但是我们在序列化器内部如何获取呢?
        # 其实我们在接口中,把序列化器类传给serializer_class的时候
        # 接口调用 get_serializer 方法的过程中, 已经把相关的参数都包含在里面了
        # 我们可以通过查看get_serializer函数, 来获取对应的验证用户.
        user_id = self.context['request'].user.id
        # 从验证过的数据中获取 sku_id
        sku_id = validated_data['sku_id']

        # 移除已经存在的本商品浏览记录
        pl.lrem("history_%s" % user_id, 0, sku_id)
        # 添加新的浏览记录
        pl.lpush("history_%s" % user_id, sku_id)
        # 只保存最多5条记录
        pl.ltrim("history_%s" % user_id, 0, constants.USER_BROWSING_HISTORY_COUNTS_LIMIT-1)

        # 执行管道中的内容
        pl.execute()

        # 返回
        return validated_data

提供的技术支持:

Redis 原生指令参数: http://redisdoc.com/index.html

Redis python 客户端方法参考: http://redis-py.readthedocs.io/en/latest/#indices-and-tables

其中, 关于上面存储方式的思考:

  1. 采用redis中哪种数据类型进行存储？ list

    答:

     string: 字符串
     history_<user_id>: '1,2,3'

 hash: 哈希   key: <field>: <value>
     history: {
         history_<user_id>: '1,2,3',
         ...
     }

 list: 列表:  key: [<value>, <value>, ...]
     histroy_<user_id>: [6, 5, 1, 2, 3]

 set: 无序集合     key: (<value>, ...)
     pass

 zset: 有序集合

  2. 添加浏览记录过程？

    答:

    history_2: [1, 2, 3]

    去重：如果用户已经浏览过该商品，先将商品id从列表中移除。lrem

    左侧加入: 保持浏览顺序。lpush

    截取: 只保留最近几条浏览记录。ltrim

后端实现

在 users / views.py 中 UserBrowsingHistoryView 视图补充 get 方法

from goods.serializers import SKUSerializer

class UserBrowsingHistoryView(CreateAPIView):
    """
    用户浏览历史记录
    """
    ...

    def get(self, request):
        """
        获取
        """
        # 获取认证用户id
        user_id = request.user.id
        # 获得redis链接到history存储区域的链接对象
        redis_conn = get_redis_connection("history")
        # 调用链接对象的lrange函数,获取某一区间的所有数据:
        history = redis_conn.lrange("history_%s" % user_id, 0, constants.USER_BROWSING_HISTORY_COUNTS_LIMIT-1)
        skus = []
        # 为了保持查询出的顺序与用户的浏览历史保存顺序一致
        # 遍历所有的历史记录, 查询出商品后依次保存到对应的列表中
        for sku_id in history:
            # 从SKU表中依次查询出各个商品
            sku = SKU.objects.get(id=sku_id)
            # 把查询出的商品保存到列表中
            skus.append(sku)

        # 对列表进行序列化操作
        # 因为列表中保存的都是从数据库中查询的商品
        # 商品需要进行序列化才能转为 json
        s = SKUSerializer(skus, many=True)
        # 返回序列化的数据
        return Response(s.data)

定义商品的序列化器:

我们需要对列表中存储的商品进行序列化, 所以需要在 goods 添加 serializers.py 文件: 

 from rest_framework import serializers
from .models import SKU


class SKUSerializer(serializers.ModelSerializer):
    '''商品的序列化器'''

    class Meta:
        model = SKU
        # 以下字段仅用于序列化使用
        fields = ('id', 'name', 'price', 'default_image_url', 'comments')
