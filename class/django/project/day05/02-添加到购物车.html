添加购物车接口

接口设计

请求方式 ： POST /cart/

请求参数： JSON 或 表单

参数 类型 是否必须 说明
sku_id int 是 商品sku id
count int 是 数量
selected bool 否 是否勾选，默认勾选

返回数据： JSON

参数 类型 是否必须 说明
sku_id int 是 商品sku id
count int 是 数量
selected bool 是 是否勾选，默认勾选

访问此接口，无论用户是否登录，前端请求都需携带请求头 Authorization，由后端判断是否登录

后端实现

因为前端可能携带 cookie，为了保证跨域请求中，允许后端使用 cookie，确保在配置文件设置如下:

# 允许白名单中的 host 跨域请求时携带 cookie
CORS_ALLOW_CREDENTIALS = True
第一步: 创建序列化器

在 carts / serialziers.py 中创建序列化器

class CartSerializer(serializers.Serializer):
    """
    购物车数据序列化器
    """
    # 添加加入购物车的三个字段, 这三个字段和模型没有关系,只是为了保存到购物车中
    # 添加的字段, 所以单独定义: 
    sku_id = serializers.IntegerField(label='sku id ', min_value=1)
    count = serializers.IntegerField(label='数量', min_value=1)
    selected = serializers.BooleanField(label='是否勾选', default=True)

    # 校验传入的数据:
    def validate(self, attr):
        try:
            # 查看传入的 sku_id 在商品中是否存在:
            sku = SKU.objects.get(id=attr['sku_id'])
        except SKU.DoesNotExist:
            # 如果不存在, 抛出异常:
            raise serializers.ValidationError('商品不存在')

        # 判断传入的数量是否超出了库存量, 如果超出,抛出异常:
        if attr['count'] > sku.stock:
            raise serializers.ValidationError('商品库存不足')

        # 检验都合格, 则返回 
        return attr
第二步: 编写视图：

在 carts / views.py 中创建视图

class CartView(APIView):
    """
    购物车
    """

    def post(self, request):
        """
        购物车记录添加
        """
        # 1. 获取参数(sku_id, count, selected)并进行参数校验
        serializer = CartSerializer(data=request.data)
        serializer.is_valid(raise_exception=True)

        # 从校验过的数据中获取:
        sku_id = serializer.validated_data['sku_id']
        count = serializer.validated_data['count']
        selected = serializer.validated_data['selected']

        # 获取登录用户
        user = request.user

        # 判断用户是否登录过
        if  user.is_authenticated:
            # 如果用户登陆过, 则进入这里: 
            # 调用 redis 保存用户的购物车记录
            # 获取 cart 的存储区域链接对象
            redis_conn = get_redis_connection('cart')

            # 保存购物车中商品及数量
            # 我们这里调用的是 hincrby 方法, 这个方法会累加增量.
            # 如果不是累加,则会创建一个新的属性和值:
            redis_conn.hincrby('cart_%s' % user.id, sku_id, count)

            # 保存购物车中商品的选中状态
            # 拼接选中状态的 key值: 
            # 判断是否勾选: 
            if selected:
                # 如果勾选, 则向 set 中添加元素
                # 我们不需要关注元素是否重复, 因为 set 中元素是唯一的.
                redis_conn.sadd('cart_selected_%s' % user.id, sku_id)

            # 3. 返回应答，保存购物车记录成功
            return Response(serializer.data, status=status.HTTP_201_CREATED)

        else:
            # 如果用户没有登录, 则进入这里: 
            # 我们需要把当前购物车中的数据保存到 cookie 中
            # 获取客户端发送的cookie信息
            cookie_cart = request.COOKIES.get('cart')

            # 判断 cookie 是否存在: 
            if cookie_cart:
                # 如果存在, 对 cookie 中的数据进行解析:
                # 先使用 base64解码, 然后 pickle 解析: 
                cart_dict = pickle.loads(base64.b64decode(cookie_cart))
            else:
                # 如果没有发送 cookie 则我们设置 cart_dict 为空
                cart_dict = {}

            # 我们判断用户之前是否将该商品加入过购物车, 如果加入过
            # 则只需要让数量增加即可. 
            # 如果没有存在过, 则需要创建, 然后增加: 
            # 形式如下所示:  
            # {
            #     '<sku_id>': {
            #         'count': '<count>',
            #         'selected': '<selected>',
            #     },
            #     ...
            # }
            if sku_id in cart_dict:
                count += cart_dict[sku_id]['count']

            cart_dict[sku_id] = {
                'count': count,
                'selected': selected
            }

            # 对 cart_dict 数据进行处理
            cart_data = base64.b64encode(pickle.dumps(cart_dict)).decode()

            # 创建 response 对象
            response = Response(serializer.data, status=status.HTTP_201_CREATED)
            # 使用 response 对象设置 cookie
            response.set_cookie('cart', cart_data, max_age=constants.CART_COOKIE_EXPIRES)
            # 返回
            return response

在 carts 中新建 constants.py 常量文件

# 购物车 cookie 的有效期
CART_COOKIE_EXPIRES = 365 * 24 * 60 * 60

注意：因为前端请求时携带了 Authorization 请求头（ 主要是 JWT ），而如果用户未登录，此请求头的 JWT 无意义（没有值），为了防止 REST framework 框架在验证此无意义的 JWT 时抛出401异常，在视图中需要做两个处理

  • 重写 perform_authentication( ) 方法，此方法是 REST framework 检查用户身份的方法
  • 在获取 request.user 属性时捕获异常，REST framework 在返回 user 时，会检查Authorization 请求头，无效的 Authorization 请求头会导致抛出异常
