Introduction

保存邮箱并发送验证邮件

学习目标:

  • 能够定义前端发送的请求邮箱申请
  • 定义接口, 完成邮箱发送功能

作用:

  • 能够让用户设置邮箱, 验证邮箱,并在用户信息页面显示验证信息

接口设计：

请求方式：PUT /email/

请求参数： JSON 或 表单

默认还有一个参数也会发给后端, 这里没有写出来, 是客户端的 JWT token

参数 类型 是否必须 说明
email str 是 Email邮箱

这里需要注意: email 是必传的参数

返回数据： JSON

返回值 类型 是否必须 说明
id int 是 用户id
email str 是 Email邮箱

视图原型:

# url(r'^email/$', views.EmailView.as_view()),  # 设置邮箱访问 url
class EmailView(UpdateAPIView):
    # 1. 接收用户发过来的数据(邮箱)
    # 2. 将邮箱更新到 User 表中: 我们的更新操作可以由序列化器来完成.
    # 3. 发送激活邮件到用户的邮箱
    # 4. 返回响应
    pass

接口实现:

在 users / views.py 中创建新视图

用于保存用户的邮箱信息，注意需要用户登录通过认证后。

from rest_framework.permissions import IsAuthenticated

# url(r'^email/$', views.EmailView.as_view()),  # 设置邮箱访问 url
class EmailView(UpdateAPIView):
    # 用户保存邮箱信息时调用的接口

    # 我们验证过 JWT token 后, 如果验证通过, 才可以设置当前视图
    permission_classes = [IsAuthenticated]
    # 设置序列化器类: 我们上面定义的序列化器:
    serializer_class = serializers.EmailSerializer

    # 返回当前访问的用户信息:
    # 这个方法必须重写:
    def get_object(self, *args, **kwargs):
        return self.request.user

序列化器实现:

在 users / serializers.py 中新建序列化器

用户验证用户提交的邮箱信息。

from rest_framework import serializers
from .models import User

class EmailSerializer(serializers.ModelSerializer):
    # 定义邮箱的序列化器: 
    # 主要用户接收用户发过来的邮箱
    # 并且把邮箱信息更新到对应的User上

    # 设置:
    class Meta:
        # 当前序列化器类针对的是 User 模型类
        model = User
        # 里面添加的字段有 id 和 email
        fields = ('id', 'email')
        # 配置额外参数: email 需要时必传的字段.
        extra_kwargs = {
            'email': {
                'required': True
            }
        }

    # 因为我们要把 email 更新到对应的 User 上去
    # 所以我们需要重写 update 函数.
    # 这里的 instance 指的是上面设置 model = User 中的: User的对象
    def update(self, instance, validated_data):
        # 我们把用户发过来的信息(validated_data)中的email拿出来
        # 更新到 User 的 email字段中:
        instance.email = validated_data['email']
        # 保存:
        instance.save()
        # 返回保存过之后的对象

        # TODO: 发送邮件进行激活, 下面会做, 这里先不看

        return instance

7.1 设置发送出去进行激活验证的 url

邮箱的激活链接是用户点击时会访问的网址，我们让用户点击时进入到 success_verify_email.html 页面

在 users / models.py 文件中, 我们创建新的函数, 进行 url 的拼接:

为什么把 url 的拼接放在 models.py 文件, 主要是希望每一个 User 都有自己独有的对应 email

from itsdangerous import TimedJSONWebSignatureSerializer

# 定义一个生成激活邮件中的 url 地址的方法
def generate_email_verify_url(self):
    '''
    生成:
     激活邮件中的 url 地址
     '''

    # 导入 itsdangerous 的方法进行加密:
    # 1. 获取 itsdangerous 的对象:
    serializer = TimedJSONWebSignatureSerializer(SECRET_KEY, 7200)
    # 2. 设置将要被加密的数据: 我们这里安装接口, 返回 用户名 和 email
    data = {
        'id':self.id,
        'email': self.email
    }
    # 调用 dumps() 函数,对数据进行加密:
    token_bytes = serializer.dumps(data) # bytes
    # 将得到的 bytes 类型转为 str
    token = token_bytes.decode()  # str

    # 拼接对应的 url
    verify_url = 'http://www.meiduo.site:8080/success_verify_email.html?token=' + token
    # 返回生成的 url
    return verify_url

7.2 在序列化器类中, 调用上面的函数, 获取 url, 并且发送激活邮件:

我们在上面写的 update( ) 方法中,继续添加内容:

from django.core.mail import send_mail
from django.conf import settings
# 因为我们要把 email 更新到对应的 User 上去
# 所以我们需要重写 update 函数.
# 这里的 instance 指的是上面设置 model = User 中的: User的对象
def update(self, instance, validated_data):
    # 我们把用户发过来的信息(validated_data)中的email拿出来
    # 更新到 User 的 email字段中:
    instance.email = validated_data['email']
    # 保存:
    instance.save()
    # 返回保存过之后的对象


    # 激活邮件的发送:
    # 1.获取邮件中的 url 地址:
    verify_url = instance.generate_email_verify_url()
    # 2. 发送邮件:
    # 设置邮件标题:
    subject = "美多商城邮箱验证"
    # 设置邮件正文, 带有标签:
    html_message = '<p>尊敬的用户您好！</p>' \
                   '<p>感谢您使用美多商城。</p>' \
                   '<p>您的邮箱为：%s 。请点击此链接激活您的邮箱：</p>' \
                   '<p><a href="%s">%s<a></p>' % (email, verify_url, verify_url)
    # 调用发送函数, 发送邮件:
      # 
    # send_mail('邮件标题', '邮件正文(不带标签)','发件人','收件人列表','邮件正文(可带标签)')
    send_mail(subject, '', settings.EMAIL_FROM, [email], html_message=html_message)

    # 返回对象
    return instance
